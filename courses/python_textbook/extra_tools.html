Некоторые инструменты, полезные в работе фонетиста, не обладают реализацией для Python. Это могут быть инструменты для командной строки или сервисы, доступные через web-API. В предыдущих главах уже упоминалось, как обращаться к программе Praat через командную строку с помощью модуля subprocess. В этой главе модуль subprocess будет рассмотрен более подробно на примере сервиса для синтеза речи eSpeak-NG. Также в главе будут рассмотрены методы обращения к веб-сервисам с помощью библиотеки requests на примере сервисов BAS Web Services (Bavarian Archive for Speech Signals).

eSpeak-NG

Программа eSpeak-NG представляет собой легковесный кроссплатформенный синтезатор речи для большого количества языков. На компьютеры Unix (в том числе Google Colab) его легко установить через терминал, для Windows можно скачать установщик из репозитория программы. Там же можно ознакомиться с документацией и списком поддерживаемых языков. Не забудьте добавить папку, где располагается исполняемый файл, в переменную среды PATH.

%%bash
sudo apt install espeak-ng

Для синтеза eSpeak-NG использует автоматические транскрипторы, построенные на правилах. В частности, для английского языка разработан достаточно качественный транскриптор, которыйподдерживает слабые (редуцированные) формы служебных слов и явления на стыках слов (например, связывание), тогда как большинство аналогов работают с текстом как с последовательностью отдельных слов. Недостатком транскриптора является его непоследовательность в некоторых транскрипциях и обозначениях.

Для работы с программами через командную строку воспользуемся встроенным модулем subprocess. Чтобы сохранить текст, полученный через стандартный вывод, нам нужна функция check_output().

Инструменты командной строки часто принимают на вход различные аргументы и опции. Их названия предваряются двумя дефисами (или одним, если используется сокращённое название опции). Каких опций требует программа, обычно можно узнать, запустив её с опциями --help или -h: espeak-ng --help.

Воспользуемся eSpeak-NG, чтобы получить транскрипцию текста на британском варианте английского языка, записанную символом МФА. Для этого передадим параметры:

  • -v en-gb-x-rp: «голос» — британский английский (Received Pronunciation);
  • -q: «тихий» вызов — собственно синтез не будет произведён;
  • --ipa: в стандартный вывод будет напечатана транскрипция МФА (можно вместо этого передать опцию -x, чтобы получить результат в формате X-SAMPA).
  • --tie: сложные символы, состоящие из более чем одной буквы, будут объединены лигатурой \u0361 (или другим символом, который можно передать через знак =). Можно также передать параметр --sep, чтобы между знаками разных фонем стоял пробел (или другой символ, который также передаётся через =).

Зададим список с этими параметрами и передадим его на вход check_output(). Функция вернёт нам последовательность байт, которую нужно превратить в строку, раскодировав как UTF-8.

import subprocess

command = ["espeak-ng", "-v", "en-gb-x-rp", "-q", "--ipa", "--tie", '"hello world"']
res = subprocess.check_output(command).decode("utf-8")
print(res)
həlˈə͡ʊ wˈɜːld
command = ["espeak-ng", "-v", "en-gb-x-rp", "-q", "--ipa", "--sep=_", '"hello world"']
res = subprocess.check_output(command).decode("utf-8")
print(res)
_h_ə_l_ˈəʊ w_ˈɜː_l_d

Обратите внимание, что система обозначений в eSpeak-NG несколько отличается от общепринятой (например, в словаре Джона Уэллза). Ниже приведена краткая таблица отличий.

eSpeak Wells
ɹ r
ɐ (употребляется непоследовательно) ə
ɛ e
iə ɪə

Кроме этого, в некоторых случаях последовательность /əl/ считается одним звуком, а в некоторых случаях используется знак слоговости \u0329 [n̩], но также непоследовательно.

Если нам не нужен стандартный вывод, можно воспользоваться функцией subprocess.call(). Например, получим файл WAV с синтезированной речью. Функция возвращает т. н. код возврата — целое число. 0 обычно указывает на успешное выполнение процесса.

fname = "hello.wav"
command = ["espeak-ng", "-v", "en-gb-x-rp", "-w", fname, '"hello world"']
code = subprocess.call(command)
if code == 0:
    print("Success!")
else:
    print("Error!")
Success!

BAS Web Services

Веб-сервисы BAS — это достаточно большой набор сервисов для обработки речевого сигнала и смежных задач для разных языков, в который входят:

  • автоматическая транскрипция (grapheme-to-phoneme, G2P);
  • автоматическая сегментация;
  • определение значений формант;
  • автоматическое слогоделение.

С полным списком сервисов и поддерживаемых языков (среди которых английский и русский) можно ознакомиться на сайте.

Большинство сервисов доступно через web-API. API (application programming interface) — это общее название для способов общения между программами. Часто этим словом обозначают web-API — то есть API для веб-сервера.

Для обращения к API используют так называемые HTTP-запросы — структурированные сообщения. Они используют разные методы в зависимости от желаемого действия. Чаще всего это методы GET для получения данных с сервера и POST для отправки данных на сервер.

Для работы с HTTP-запросами в Python наиболее удобное средство — сторонняя библиотека requests (pip install requests). В Google Colab она установлена по умолчанию.

Кроме библиотеки requests, нам понадобится модуль xml для анализа результатов запроса, приходящих в формате xml.

import requests
import xml.etree.ElementTree as ET

Запросы к сервисам BAS отправляются на адреса вида https://clarin.phonetik.uni-muenchen.de/BASWebServices/services/<SERVICE>. Например, сервис для получения автоматической транскрипции называется runG2P. Полная документация к API BAS с указанием названий сервисов и их параметров расположена по ссылке.

baseurl = "https://clarin.phonetik.uni-muenchen.de/BASWebServices/services/"
service = "runG2P"

url = baseurl + service

Чаще всего HTTP-запрос содержит определённый набор параметров. Они могут передаваться в разном виде, и в requests для этого используются разные аргументы функций для соответствующих запросов. Эти параметры могут быть текстовыми полями или содержимым файлов (как текстовых, так и нет).

Воспользуемся сервисом runG2P, чтобы получить транскрипцию текста. Алгоритм G2P для русского языка в сервисах BAS далёк от совершенства, поэтому работу сервиса продемонстрируем на примере британского английского. Запишем желаемый текст в новый файл:

text = "Hello world!"
filename = "hello.txt"

with open(filename, "w", encoding="utf-8") as f:
    f.write(text)

Зададим необходимые параметры для сервиса runG2P. Текстовые параметры зададим в словаре data. Здесь укажем язык (британский английский), алфавит для записи результата (МФА) и формат результата (текстовый файл). Другие значения, которые можно установить этим параметрам, а также дополнительные параметры перечислены в документации по ссылке выше. В частности, МФА можно заменить на универсальный алфавит X-SAMPA ("x-sampa") или версию SAMPA, модифицированную под выбранный язык ("sampa").

data = {
    "lng": "eng-GB",
    "outsym": "ipa",
    "oform": "txt"
}

Файлы для обработки зададим в словаре files. Теперь значениями будут кортежи, в которых мы перечислим название файла, файловый объект для работы с ним и указатель на тип данных, содержащийся в файле. В нашем случае, поскольку файл текстовый, это будет строка "text/txt".

files = {
    "i": (filename, open(filename, encoding="utf-8"), "text/txt")
}

Теперь мы можем отправить POST-запрос на сервер. Ответ на запрос будет записан в переменную res.

res = requests.post(url, files=files, data=data)

В зависимости от того, к какому результату привёл запрос, у ответа будут разные коды статуса. Если запрос был успешно обработан, код будет равен 200.

В случае успеха сервис вернёт нам в качестве результата xml-файл, в котором в теге downloadLink будет содержаться временная ссылка на файл с результатом на сервере. Найдём её с помощью библиотеки xml и отправим на неё запрос GET, чтобы скачать получившийся файл.

Теперь результат хранится в атрибуте text результата второго запроса. Чтобы символы МФА были интерпретированы правильно, установим кодировку utf-8. Результат содержит фонемную транскрипцию изначального текста, где транскрипционные знаки отделены пробелами, а слова — знаком табуляции.

if res.status_code == 200:
    tree = ET.fromstring(res.text)
    res2 = requests.get(tree.find('downloadLink').text)
    if res2.status_code == 200:
        res2.encoding = "utf-8"
        print(res2.text)
    else:
        print("Произошла ошибка при получении результата!")
        print(res2.text)
else:
    print("Произошла ошибка при отправке данных!")
    print(res.text)
h ə l əʊ    w ɜː l d

Рассмотрим теперь работу с сервисами runMAUS и runMAUSBasic, которые позволяет получить границы звуков в сигнале. Первый принимает на вход готовую фонетическую транскрипцию текста и в целом допускает более гибкую настройку, второй же работает с орфографической записью и использует алгоритмы G2P.

Проиллюстрируем работу первого на примере файла cta0001.wav. Нам необходимо будет задать транскрипцию заранее. Мы можем использовать как МФА, так и X-SAMPA; для простоты воспользуемся последней. Алфавит символов для каждого языка можно посмотреть с помощью сервиса runMAUSGetInventar, передав в качестве параметра название языка. Это можно сделать прямо в браузере, перейдя по ссылке. Можно получить эту информацию и программно:

service = "runMAUSGetInventar"
url = baseurl + service

lang = "rus-RU"
params = {"LANGUAGE": lang}

res = requests.get(url, params)
res.encoding = "utf-8"
with open(f"webmaus_{lang}.tsv", "w") as f:
    for line in res.text.split("\n"):
        if not line.startswith("%"):
            f.write(line + "\n")

Получившаяся таблица содержит разные обозначения для всех звуков, определённых для данного языка. Её можно использовать для внутренней конвертации или для проверки валидности транскрипции.

Составим таблицу CSV, в которой мы укажем произношение каждого слова. Ячейки таблицы отделим друг от друга точкой с запятой ;, транскрипционные символы разделим пробелами.

csv_filename = "cta0001.csv"
wav_filename = "cta0001.wav"
res_filename = "cta0001.TextGrid"

with open(csv_filename, "w", encoding="utf-8") as f:
    f.write("юрий;j u r_j i j\n")
    f.write("трифонов;t r_j i f a n a f\n")

Теперь зададим нужные параметры (обратите внимание, что теперь их названия другие). Звук передадим в поле "SIGNAL", транскрипции — в поле "BPF". В качестве формата выходных данных выберем TextGrid.

service = "runMAUS"
url = baseurl + service

data = {
    "LANGUAGE": "rus-RU",
    "INSYMBOL": "sampa",
    "OUTFORMAT": "TextGrid",
    "OUTSYMBOL": "ipa"
}

files = {
    "SIGNAL": (wav_filename, open(wav_filename, "rb"), "audio/wav"),
    "BPF": (csv_filename, open(csv_filename), "text/csv")
}

Отправим запрос:

res = requests.post(url, files=files, data=data)
if res.status_code == 200:
    tree = ET.fromstring(res.text)
    res2 = requests.get(tree.find('downloadLink').text)
    if res2.status_code == 200:
        print(f"Запись в файл {res_filename}")
        res2.encoding = "utf-8"
        with open(res_filename, "w") as f:
            f.write(res2.text)
    else:
        print("Произошла ошибка при получении результата!")
        print(res2.text)
else:
    print("Произошла ошибка при отправке данных!")
    print(res.text)

Так будет выглядеть результат, если открыть его в программе Praat. На уровне "ORT-MAU" помещены границы слов в орфографии, на уровне "KAN-MAU" — те же слова в транскрипции, на уровне "MAU" — границы звуков. Если при загрузке файла передать параметр INSKANTEXTGRID=false (добавьте в словарь data строку "INSKANTEXTGRID": "false"), то уровень "KAN-MAU" не будет сгенерирован.

Если вы хотите каким-то образом изменить полученный TextGrid, можно передать его в TextGridTools, не записывая в файл. Для этого воспользуемся обёрткой для строк StringIO из встроенного модуля io, которая позволяет получить из строки файловый объект.

from io import StringIO
import tgt

grid = tgt.read_textgrid(StringIO(res2.text))
