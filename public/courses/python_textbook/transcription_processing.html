При работе с фонетической транскрипцией фонетист неизбежно сталкивается с необходимостью её автоматической обработки и перекодирования в разных стандартах. Самым распространённым (и, следовательно, рекомендуемым) стандартом транскрипционных записей является Международный фонетический алфавит (МФА), для каждого символа которого существует официально предписанный уникальный код в стандарте Unicode. Так, например, транскрипции, приведённые в статьях проекта Викисловарь, используют именно символы МФА. Однако по разным причинам существующие материалы, в частности, корпусы речевых данных, записанные на кафедре фонетики СПбГУ, могут использовать другие стандарты — например, основанные только на базовой латинице (как, например, корпус CORPRES) или на традиционной Щербовской системе транскрипции (например, корпусы CoRuSS и SibLing). Таким образом, фонетист должен внимательно относиться к выбору знаков фонетической транскрипции и стараться по возможности использовать корректные символы Unicode.

В этой главе приведено краткое руководство по использованию особых символов в коде программ, а также будут рассмотрены некоторые библиотеки, облегчающие работу с транскрипцией.

Международный фонетический алфавит и стандарт Unicode

Стандарт Unicode содержит коды для огромного количества символов (почти 300 000 в версии 17.0), многие из которых являются производными версиями других. Это касается и символов МФА, а следовательно, у многих символов МФА будут «дубликаты» — символы, идентичные или очень схожие с ними по начертанию. В связи с этим для ввода символов МФА рекомендуется использовать специальные клавиатуры. Онлайн-клавиатуры можно найти на официальном сайте МФА или на сайте онлайн-клавиатур typeit. Также можно установить специальные раскладки.

В коде программы можно использовать символы МФА напрямую. Однако это не всегда рекомендовано: многие диакритические символы МФА являются комбинируемыми (combining), т. е. они накладываются на предыдущий символ. Кроме того, некоторые символы могут не поддерживаться шрифтом среды разработки. В связи с этим более надёжным способом является указание кодов Unicode. Для этого нужно использовать специальную escape-последовательность \u, после чего набрать код Unicode — 4 шестнадцатеричные цифры (\uxxxx). Если в коде более 4 цифр, необходимо использовать следующий формат: \Uxxxxxxxx. Коды доступны в специальных таблицах, например, на соответствующей странице Википедии.

# LATIN SMALL LETTER I WITH STROKE
# (close central unrounded vowel):
print("\u0268")
# t + LATIN SMALL LETTER ESH + COMBINING DOUBLE INVERTED BREVE
# (voiceless postalveolar affricate):
print("t\u0361\u0283")
ɨ
t͡ʃ

Поскольку такие коды неудобочитаемы, рекомендуется оставлять комментарии с названием или описанием символа.

Шестнадцатеричный код символа можно получить также внутренними средствами Python. Например, можно получить его целочисленное представление с помощью встроенной функции ord(), которая принимает на вход строку с одним символом и возвращает целое число — его номер в Unicode (существует также обратная ей функция chr(), которая принимает на вход целое числе и возвращает строку с соответствующим символом Unicode). Полученное число можно перевести в шестнадцатеричную систему счисления с помощью встроенной функции hex().

print(hex(ord("ɨ")))
0x268

Также можно использовать строковый метод .encode(), который принимает на вход название кодировки и возвращает последовательность байт. Передадим в него специальную кодировку "unicode_escape":

print("ɨ".encode("unicode_escape"))
b'\\u0268'

Наконец, стоит учитывать, что всё описанное выше касается только программ, исходный код которых сохранён в кодировках Unicode (например, UTF-8). Если код сохранён в какой-либо другой кодировке (хотя это не рекомендуется), использование особых символов будет невозможно или существенно ограничено.

Триграфы Praat

Для хранения символов, не входящих в таблицу ASCII (символов МФА, кириллицы и других алфавитов), Praat пользуется своей собственной системой: каждому символу, не входящему в ASCII, сопоставляется т. н. триграф. Под триграфом понимается последовательность из обратного слеша \ и двух символов ASCII. Например, символу ɨ соответствует триграф \i-.

https://www.fon.hum.uva.nl/praat/manual/TextGrid_file_formats.html
https://www.fon.hum.uva.nl/praat/manual/Special_symbols.html

Всего в Praat определено несколько сотен таких триграфов, набор периодически расширяется (всего их возможно несколько тысяч, что, конечно, гораздо меньше, чем количество символов, определённых в Unicode).

Файлы .TextGrid могут содержать как один вариант записи, так и другой. Для конвертации рекомендуется использовать саму программу Praat (вручную, написав скрипт или через библиотеку parselmouth). Однако здесь система будет рассмотрена детально в информационных целях. Напишем свою программу для конвертации из триграфов в Unicode и обратно. Для этого посмотрим на ту часть исходного кода Praat, которая отвечает за конвертацию.

# скачаем файлы исходного кода
!wget https://raw.githubusercontent.com/praat/praat/master/kar/longchar.cpp
!wget https://raw.githubusercontent.com/praat/praat/master/kar/UnicodeData.h
with open("UnicodeData.h") as f:
    lines = f.readlines()

# здесь хранятся коды символов Unicode в шестнадцатеричном представлении
unicode_vals = {}
for line in lines:
    if not line.startswith("#define"):
        continue
    _, name, val = line.strip().split()
    if not name.startswith("UNICODE"):
        continue
    unicode_vals[name] = chr(int(val, 16))
import re

with open("longchar.cpp") as f:
    lines = f.readlines()

trigraph2unicode = {}

# здесь хранится таблица соответствий
for line in lines:
    line = line.replace("\\'", "'")
    line = line.replace('\\"', '"')
    # напишем регулярное выражение, которое ищет в строке таблицы символы,
    # входящие в триграф, и название символа Unicode
    m = re.search(r"('[^,]+'), ?('[^,]+'),.+(UNICODE_\w+)", line)
    if m is None:
        continue
    ch1, ch2, name = m[1][1:-1], m[2][1:-1], m[3]
    if ch2 == " ":
        continue
    trigraph2unicode["\\" + ch1 + ch2] = unicode_vals[name]
# сделаем словарь для обратной конвертации
unicode2trigraph = {j: i for i, j in trigraph2unicode.items()}
# проверим:
trigraph2unicode["\\O\""]
'Ö'
unicode_string = "bɨl tʲixʲij sʲerɨj vʲet͡ʃʲir"
trigraph_string = unicode_string.translate(str.maketrans(unicode2trigraph))
print(trigraph_string)
b\i-l t\^jix\^jij s\^jer\i-j v\^jet\li\sh\^jir

Библиотека phonecodes

Существует несколько библиотек, которые помогают работать с фонетической транскрипцией. Одна из них — phonecodes, которая позволяет конвертировать транскрипцию из одной системы в другую.

Выведем поддерживаемые системы транскрипции:

from phonecodes import phonecodes
print(phonecodes.CODES)
{'buckeye', 'xsampa', 'disc', 'ipa', 'callhome', 'arpabet'}

IPA — это Международный фонетический алфавит. X-SAMPA — это способ записи МФА с помощью только символов ASCII, предложенный Дж. Уэллзом.

phonecodes.convert("ˈbɨl ˈtʲixʲij ˈsʲerɨj ˈvʲet͡ʃʲir", "ipa", "xsampa")
'"b1l "t_jix_jij "s_jer1j "v_jet͡S_jir'

Эквивалентное использование:

phonecodes.ipa2xsampa("ˈbɨl ˈtʲixʲij ˈsʲerɨj ˈvʲet͡ʃʲir")
'"b1l "t_jix_jij "s_jer1j "v_jet͡S_jir'

Обратите внимание, что некоторые символы не конвертируются — например, дуги, соединяющие аффрикаты и двойные артикуляции. Их можно заменить нижним подчёркиванием:

phonecodes.ipa2xsampa("ˈbɨl ˈtʲixʲij ˈsʲerɨj ˈvʲet͡ʃʲir").replace("\u0361", "_")
'"b1l "t_jix_jij "s_jer1j "v_jet_S_jir'

Другая система транскрипции, представленная в phonecodes, — ARPABET. Это система для английского языка, также целиком основанная на символах ASCII.

phonecodes.convert("ð ɪ s ɪ z ə t ˈɛ s t", "ipa", "arpabet")
'DH IH S IH Z AH0 T EH1 S T'

Стоит заметить, что ARPABET может выглядеть немного по-разному в разных источниках. В частности, в корпусе TIMIT версия ARPABET отличается от базовой. Версия, представленная в phonecodes, подойдёт, например, чтобы работать со словарём CMU Pronouncing Dictionary.

phonecodes.convert("G R IY N", "arpabet", "ipa")
'ɡ ɹ i n'

Библиотека segments

Библиотека segments была разработана С. Мораном и Р. Форкелем для облегчения операций с орфографией разных языков. Её основная функция — токенизация, т. е. деление на токены — в данном случае это графемы.

Для токенизации инициализируем экземпляр класса Tokenizer и вызовем его как функцию, передав первым аргументом строку с транскрипцией, а вторым — именованный аргумент ipa=True. Объект вернёт нам строку, в которой все символы МФА отделены друг от друга пробелами с учётом диакритических знаков. На месте пробелов в исходной строке будут стоять знаки решётки #.

from segments import Tokenizer

tok = Tokenizer()
tokenized = tok("ˈbɨl ˈtʲixʲij ˈsʲerɨj ˈvʲet͡ʃʲir", ipa=True)
print(tokenized)
ˈb ɨ l # ˈtʲ i xʲ i j # ˈsʲ e r ɨ j # ˈvʲ e t͡ʃʲ i r

Изменить символы-разделители можно с помощью специальных аргументов segment_separator (разделитель знаков) и separator (разделитель слов):

tokenized = tok(
    "ˈbɨl ˈtʲixʲij ˈsʲerɨj ˈvʲet͡ʃʲir",
    ipa=True,
    segment_separator="\t",
    separator="\n"
)
print(tokenized)
ˈb  ɨ   l
ˈtʲ i   xʲ  i   j
ˈsʲ e   r   ɨ   j
ˈvʲ e   t͡ʃʲ    i   r

Для работы с другими системами можно задать т. н. орфографический профиль — таблицу графем и (опционально) их соответствий в других системах (например, МФА или X-SAMPA). Такие таблицы рекомендуется задавать в текстовых файлах (например, CSV).

from segments import Profile

prf = Profile(
    {
        "Grapheme": "ab",
        "mapping": "x",
    },
    {
        "Grapheme": "cd",
        "mapping": "y",
    }
)
print(prf)
Grapheme    mapping
ab  x
cd  y

Профиль передаётся в аргумент profile при инициализации токенизатора, после чего можно переводить символы из одной колонки таблицы в другую, задавая название этой колонки в аргументе column.

tok = Tokenizer(profile=prf)
tokenized = tok("abcd", column="mapping")

print(tokenized)
x y

По умолчанию символы, отсутствующие в таблице, будут заменяться на �. Можно прописать специальное правило для работы с такими символами. Например, передадим в такое правило лямбда-функцию, возвращающую свой аргумент. Тогда неизвестные символы останутся на месте:

tok = Tokenizer(profile=prf, errors_replace=lambda x: x)
tokenized = tok("abcdq", column="mapping")

print(tokenized)
x y q

segments также обладает интерфейсом для взаимодействия через командную строку. Подробнее можно прочитать в документации или в книге авторов "The Unicode cookbook for linguists: Managing writing systems using orthography profiles".

Библиотека panphon

Ещё одна интересная библиотека — panphon, которая позволяет конвертировать символы МФА в наборы бинарных фонологических признаков.

import panphon._panphon as panphon
ft = panphon.FeatureTable()

Набор признаков, которым пользуется panphon, таков:

syl [±syllabic]. Is the segment the nucleus of a syllable?
son [±sonorant]. Is the segment produced with a relatively unobstructed vocal tract?
cons [±consonantal]. Is the segment consonantal (not a vowel or glide, or laryngeal consonant)?
cont [±continuant]. Is the segment produced with continuous oral airflow?
delrel [±delayed release]. Is the segment an affricate?
lat [±lateral]. Is the segment produced with a lateral constriction?
nas [±nasal]. Is the segment produced with nasal airflow?
strid [±strident]. Is the segment produced with noisy friction?
voi [±voice]. Are the vocal folds vibrating during the production of the segment?
sg [±spread glottis]. Are the vocal folds abducted during the production of the segment?
cg [±constricted glottis]. Are the vocal folds adducted during the production of the segment?
ant [±anterior]. Is a constriction made in the front of the vocal tract?
cor [±coronal]. Is the tip or blade of the tongue used to make a constriction?
distr [±distributed]. Is a coronal constriction distributed laterally?
lab [±labial]. Does the segment involve constrictions with or of the lips?
hi [±high]. Is the segment produced with the tongue body raised?
lo [±low]. Is the segment produced with the tongue body lowered?
back [±back]. Is the segment produced with the tongue body in a posterior position?
round [±round]. Is the segment produced with the lips rounded?
tense [±tense]. Is the segment produced with an advanced tongue root.

Функция fts() вернёт объект класса Segment, содержащий набор признаков для одного звука (или пустой словарь, если поданная строка не является символом МФА):

ft.fts("tʰ")
<Segment [-syl, -son, +cons, -cont, -delrel, -lat, -nas, -strid, -voi, +sg, -cg, +ant, +cor, -distr, -lab, -hi, -lo, -back, -round, -velaric, 0tense, -long, 0hitone, 0hireg]>
ft.fts("ы")
{}

Значения признаков можно извлекать из объекта по ключу, как из словаря. Возможных значений три: 1 (признак присутствует), −1 (признак отсутствует), 0 (признак не определён).

ft.fts("tʰ")["cons"]
1

Функция word_fts() сгенерирует список наборов признаков для каждого звука в слове:

ft.word_fts('pʲãk')
[<Segment [-syl, -son, +cons, -cont, -delrel, -lat, -nas, -strid, -voi, -sg, -cg, +ant, -cor, 0distr, +lab, +hi, -lo, -back, -round, -velaric, 0tense, -long, 0hitone, 0hireg]>,
 <Segment [+syl, +son, -cons, +cont, -delrel, -lat, +nas, -strid, +voi, -sg, -cg, 0ant, -cor, 0distr, -lab, -hi, +lo, +back, -round, -velaric, +tense, -long, 0hitone, 0hireg]>,
 <Segment [-syl, -son, +cons, -cont, -delrel, -lat, -nas, -strid, -voi, -sg, -cg, -ant, -cor, 0distr, -lab, +hi, -lo, +back, -round, -velaric, 0tense, -long, 0hitone, 0hireg]>]

При работе с библиотекой нужно учитывать, что сама концепция бинарных фонологических признаков достаточно спорна, особенно в таком универсальном (языконезависимом) представлении. Это приводит к неточностям в представлении: так, [a] и [ɑ] отличаются здесь только тем, что у [a] признак delrel (delayed release) отсутствует, а у [ɑ] — не определён. Чтобы убедиться в этом, используем метод .differing_specs().

f1, f2 = ft.word_fts('aɑ')
f1.differing_specs(f2)
['delrel']
print(f1["delrel"], f2["delrel"])
-1 0
